# Конфигурация привычек в формат YAML

Отлично, добавляем возможность гибкой настройки напоминаний в конфигурацию.

## Расширенный формат YAML с поддержкой напоминаний

```yaml
habits:
- name: "Утренняя зарядка"
  type: "simple"
  reminder:
  time: "08:00"  # Напоминание в конкретное время
  days: ["mon", "tue", "wed", "thu", "fri"]  # Напоминание только по будням

- name: "Отжимания"
  type: "counter"
  target: 30
  reminder:
  time: "18:00"  # Напоминание вечером каждый день

- name: "Проснуться в 6 утра"
  type: "time"
  target_time: "06:00"
  reminder:
  days: ["sat", "sun"]  # Напоминание только по выходным

- name: "Чтение книги"
  type: "simple"
  reminder:
  time: "21:00"  # Напоминание каждый день перед сном
  days: ["mon", "wed", "fri"]  # Напоминание в определённые дни недели
```

### Объяснение новых параметров:

- reminder:
- time — конкретное время для напоминания (формат HH:MM).
- days — список дней недели, когда нужно напоминать (mon, tue, wed, thu, fri, sat, sun).
- Если указаны оба параметра (time и days), напоминание будет срабатывать в указанное время только в определённые дни недели.

### Пример использования:

1. В 8 утра по будням бот будет напоминать о зарядке:
```
Напоминание: не забудь сделать утреннюю зарядку!
```

2. Вечером в 18:00 бот напомнит о необходимости отжиманий:
```
Напоминание: время отжиматься! Цель — 30 раз.
```

3. По выходным бот напомнит проснуться вовремя:
```
Напоминание: не забудь проснуться до 6 утра!
```


Реализация напоминаний:

- Мы можем запланировать регулярную проверку напоминаний (например, каждые 10-15 минут) 
    и проверять текущее время и день недели, чтобы отправить сообщение пользователю.
- Это будет реализовано через cron-задачу в боте.
